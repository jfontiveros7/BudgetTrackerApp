# Packaging Guide

## Gumroad

Recommended package contents:

- full source code
- README
- installation steps
- SQL schema
- demo seed data
- screenshots
- license terms
- support/contact note

Suggested archive structure:

- `source/`
- `documentation/`
- `screenshots/`
- `license/`

## CodeCanyon

Recommended package contents:

- clean source code
- installation documentation
- feature list
- changelog
- SQL schema and demo data
- screenshots
- support scope
- final marketplace-ready license terms

## Before Selling

- remove secrets and real keys
- confirm `.env` examples are safe
- test fresh install on a clean machine
- verify demo login works
- verify screenshots match the shipped product
- verify README steps are complete

## Suggested Deliverables

- product zip
- documentation zip or pdf
- screenshots folder
- support/contact instructions
